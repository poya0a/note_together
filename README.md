# NoteTogether — 실시간 협업 에디터

        하나의 링크로 여러 사용자가 동시에 문서를 편집하고,  
        모든 변경 사항을 실시간으로 공유할 수 있는 실시간 협업 문서 편집기

**배포 주소**  
https://note-together-kappa.vercel.app

---

## 주요 기능

**실시간 협업 편집**
- 여러 사용자가 동시에 문서 편집 가능
- 입력 지연 없이 즉시 반영되는 실시간 동기화
- CRDT 기반으로 충돌 없는 편집 처리

**리치 텍스트 에디터**
- TipTap 기반 리치 텍스트 에디터 구현
- 제목 영역과 본문 영역 분리 구성
- 직관적인 문서 작성 UX 제공

**사용자 상태 표시 (Awareness)**
- 접속 중인 사용자 커서 및 포인터 실시간 표시
- 현재 누가 어디를 편집 중인지 시각적으로 확인 가능

**문서 관리**
- 하나의 링크로 문서 공유 및 접속
- 문서 삭제 시 모든 접속자에게 실시간 알림
- 문서 삭제와 동시에 세션 종료 처리

---

## 기술 스택

**Frontend**: Next.js, React, TypeScript  
**Backend**: Supabase,  
**Editor & Realtime**: TipTap, Y.js (CRDT), Hocuspocus  
**Styling**: SCSS   
**Deployment**: Vercel

---

### CRDT 기반 실시간 동기화 구현
        Y.js를 활용해 중앙 서버 의존 없이 문서 상태 동기화
        다중 사용자 동시 편집 환경에서도 데이터 충돌 없이 병합
        실시간 협업 서비스의 핵심 동작 원리에 대한 이해도 향상

### TipTap 커스터마이징 경험
        기본 에디터 기능 위에 프로젝트 요구사항에 맞게 확장
        제목과 본문을 분리해 각각 독립적으로 동기화
        Y.Map과 Y.XmlFragment를 활용한 문서 구조 설계

### 실시간 UX를 고려한 상태 관리
        사용자 접속 상태 및 커서 정보를 Awareness로 관리
        Zustand를 활용해 에디터 외부 UI 상태를 명확하게 분리
        실시간 데이터 흐름에서도 UI 혼란을 최소화하는 구조 설계

### 협업 서비스에서의 예외 상황 처리
        문서 삭제 시 모든 사용자에게 즉시 알림 전송
        세션 강제 종료 처리로 데이터 불일치 방지
        실시간 서비스에서 발생할 수 있는 엣지 케이스 대응 경험

## 프로젝트 목적

- 실시간 협업 서비스의 구조와 동작 방식 이해
- CRDT 기반 데이터 동기화 경험
- 복잡한 상태 흐름을 가진 프론트엔드 애플리케이션 설계 역량 강화
